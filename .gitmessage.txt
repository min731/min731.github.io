# type (필수): commit의 유형
#   - build: 빌드 시스템 변경 (ex. 빌드 도구, 패키지 매니저 변경)
#   - chore: 기타 변경사항 (코드 수정 없음)
#   - ci: CI/CD 관련 변경
#   - docs: 문서 수정
#   - feat: 새로운 기능 추가
#   - fix: 버그 수정
#   - perf: 성능 개선
#   - refactor: 코드 리팩토링 (기능 변화 없음)
#   - revert: 이전 커밋 되돌리기
#   - style: 코드 스타일 수정 (공백, 세미콜론 등)
#   - test: 테스트 코드 추가/수정

# scope (선택): 변경된 기능이나 모듈을 명확히 지정
# subject (필수): 50자 이내로 요약 (소문자로 시작, 명령형 사용)
# body (선택): 변경 내용 설명 (어떤 변경을 했고, 왜 했는지)
# footer (선택): BREAKING CHANGE (큰 변경 사항) 또는 issue tracker ID 참조
